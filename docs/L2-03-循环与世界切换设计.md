## 1. 文档目的

把"按 N 进入下一轮"从机械的染银操作升级为**真正的世界切换**：每轮加载一张不同的 GLB，让玩家直观体验"地图变了"的认知崩塌。同时引入 Phantoms（人形）几何作为"过去的自己"的视觉化身。

## 2. 五轮 → 四张地图映射

| 轮次 | GLB 文件 | 角色 |
|------|----------|------|
| 1 | `scene.glb` | 正常世界，玩家熟悉环境 |
| 2 | `scene.glb` | **同一张地图**，制造"还是熟悉的"安全感 |
| 3 | `scene_loop3.glb` | 开始 glitchy——细节扭曲、出现第一批 phantoms |
| 4 | `scene_loop4.glb` | 怪——大件错位、出现凭空几何、phantoms 增多 |
| 5 | `scene_loop5.glb` | 诡异——世界几何明显异化、phantoms 扭曲变形 |

**为什么 1+2 共用**：玩家在第二轮会本能预期"还是这张图"，到第三轮的突变冲击力翻倍。

## 3. 文件布局

```
content/levels/earth_return_01/
  ├── scene.glb          # 轮 1+2
  ├── scene_loop3.glb    # 轮 3
  ├── scene_loop4.glb    # 轮 4
  └── scene_loop5.glb    # 轮 5
```

## 4. 加载策略

**MVP 阶段：启动时全部加载**。

理由：
- 单张地图几 MB，4 张 < 30MB，内存代价可忽略
- 运行时切换零延迟、零卡顿
- 实现极简：`Vec<World>`，切换只改索引

**正式阶段**：开场 Loading 场景显式播放"系统启动"叙事文本（参考 L0 H3 系统启动日志原文）+ 进度条，加载在叙事播放期间完成。详细见 §8。

## 5. 切换机制

```
GameApp 持有:
  worlds: Vec<World>              # 启动时全部 load
  phase: u32                       # 当前轮次 [1..=5]
  world_index_for_phase(p) -> usize  # phase→world 映射

按 N（或能量耗尽，未来）:
  sonar.advance_loop()          # 当前点云染银（已实现）
  phase += 1
  if phase > 5: phase = 1       # 或触发结局
  let new_idx = world_index_for_phase(phase)
  if new_idx != current_idx:
    renderer.reload_world(&worlds[new_idx])  # 重建 depth pipeline
    current_idx = new_idx
  player.respawn(worlds[current_idx].spawn())
```

## 6. 残留点云跨轮持久化

**铁律：所有 GLB 共享世界坐标系**。Blender 中 phase3/4/5 的地图在**同样的全局坐标**下建模，原点不变。

后果：染银后的点云不需要做任何坐标变换——它们的世界坐标在新地图里仍然有意义。玩家可能扫到一面"上一轮存在但这轮没了"的银色墙轮廓飘在空中——这正是"过去记忆与当下世界不符"的字面化，是主题武器，不是 bug。

## 7. Phantoms（人形）几何

### 7.1 引入

Phantoms 是**专门用作"过去的自己/异形回声"视觉化身**的特殊 mesh，不参与碰撞、不参与玩家可走判定，**只参与声呐 raycast**——扫到就显形，显形颜色为**红色**。

### 7.2 Blender 端约定

- 新 collection：`Phantoms`
- 命名前缀：`P_humanoid_<id>` / `P_phantom_<id>`（前缀 `P_`）
- 不进 `Render` 也不进 `Collision`——独立分类
- Mesh 内容由 PA 在 Blender 中自由建模（低面人形为 MVP 起点）

### 7.3 运行时

- 内容管线（`src/content/mod.rs`）按 `P_` 前缀把 Phantoms mesh 分类进 `LoadedLevel.phantom_tris`
- World 持有独立的 `phantom_tris: Vec<Tri>`，**不进 collision 网格**
- 声呐 `raycast` 扩展：先对 Phantoms 求交、再对 Render 求交，取最近命中。命中 Phantoms 时返回特殊 surface 标记 `Surface::Phantom`
- 颜色规则：`Surface::Phantom` → 红色（`Color::new(1.0, 0.15, 0.18, 1.0)` 起点，可调）
- 散射规则：不生成地面/天花板附带点——phantom 形状应该清晰、独立、不被环境点云稀释

### 7.4 主题表达递进

| 轮次 | Phantom 角色 |
|------|--------------|
| 1, 2 | 无 phantoms |
| 3 | 少量静态人形，深处地点 |
| 4 | 增多，可能放置在"上一轮玩家停留过的关键位置" |
| 5 | 扭曲变形（多头/断肢/异常拉伸）|

第 5 轮的扭曲变形仍是手工建模，不是程序化。

## 8. 开场 Loading 场景（后续阶段）

**MVP 不做**：当前 4 张 GLB 在 `GameApp::new()` 中同步加载，启动延迟 ~1 秒可接受。

**正式做法**（待实现）：
- `AppState::Loading { progress, message }` → `AppState::Playing`
- 异步加载 GLB（macroquad 已支持 async）
- 屏幕中央播放 L0 H3 启动日志文本，进度条同步
- 加载完毕淡入第一张地图

## 9. 失败处理

- 启动时任一 GLB 缺失 → 该 phase 回退到上一张可用 GLB，console 打印警告（避免开发期阻塞）
- 所有 GLB 都缺失 → 回退代码盒子房间（现有兜底逻辑）

## 10. 决策记录

### D-001
- 决策：4 张 GLB，轮 1+2 共用第一张
- 理由：制造"熟悉感被偷走"的反转节奏，PA 设计直觉
- 失效条件：若改为每轮独立扭曲

### D-002
- 决策：启动时全部加载，运行时零开销切换
- 理由：4 张 GLB 总大小 < 30MB，内存代价可忽略；切换时机不应有任何加载卡顿
- 失效条件：若 GLB 规模扩大 10×

### D-003
- 决策：所有 GLB 共享世界坐标系，残留点云不变换
- 理由：让"过去的记忆与当下世界不符"成为玩家可感知的主题武器
- 失效条件：若设计上需要轮次之间有空间位移

### D-004
- 决策：Phantoms 独立 collection + `P_` 前缀
- 理由：与 Render/Collision 语义清晰分离；运行时按前缀分类无需侧车
- 失效条件：若需要更精细的"phantom 类型"分类（如 distorted/humanoid/abstract）
