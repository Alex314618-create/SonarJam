## 1. 文档目的

本文档用于记录当前已批准的技术事实与正式生产入口。

本文档是可变的技术基线登记表，不是治理宪法。所有修改必须遵守 [L0-技术宪法.md](L0-技术宪法.md) 的变更与审批规则。

| 领域 | 当前批准基线 | 备注 |

## 4. 决策记录

### RTB-001

- 决策：目标平台定为 `Windows PC`
- 理由：当前比赛交付路径最短、适配成本最低
- 备选方案：未记录为正式桌面外平台；移动平台与 Web 未被当前交付目标采用
- 未选原因：额外适配、输入方案、打包链路与验证成本更高
- 失效条件：若比赛或发布要求明确改变目标平台

### RTB-002

- 决策：开发语言定为 `Rust`
- 理由：已明确选择 Rust 作为本次项目唯一实现语言
- 备选方案：其他语言与引擎路线未纳入当前批准基线
- 未选原因：与当前已确认开发偏好和执行路径不一致
- 失效条件：若 `PA` 批准更改实现语言

### RTB-003

- 决策：运行时框架定为 `macroquad`
- 理由：当前阶段需要轻量、可快速验证、适合短周期交付的框架
- 备选方案：其他 Rust 运行时框架未纳入当前批准基线
- 未选原因：当前阶段不需要更重的框架复杂度
- 失效条件：若 `PA` 批准更改运行时框架

### RTB-004

- 决策：正式地图生产入口定为 `Blender`
- 理由：空间控制与迭代效率优于纯代码拼装，适合作为正式内容入口
- 备选方案：纯代码拼装或其他 DCC 工具未纳入当前正式入口
- 未选原因：空间控制与可视化迭代能力不足，或当前流程未被批准
- 失效条件：若正式内容生产入口被替换

### RTB-005

- 决策：MVP 阶段采用**代码定义几何**构建主场景，`Blender -> GLB` 内容入口推迟启用
- 理由：48 小时交付下优先验证核心声呐手感与 macroquad 3D 可行性（spike 优先）；GLB 导入器属重型前置基础设施，会推迟手感反馈
- 备选方案：从一开始就走 Blender→GLB 正式管线
- 未选原因：前期基础设施成本高，且核心玩法风险未先行验证
- 范围：本决策**临时偏离 RTB-004**。`Blender` 作为正式内容入口的地位保留，仅推迟启用；手感参照为 `C:\Users\ROG\Desktop\sonar` C 原型
- 失效条件：核心手感验证通过、需要正式关卡内容时，恢复 RTB-004 的 `Blender` 管线
- 批准来源：Product Authority（2026-05-29 对话确认）
